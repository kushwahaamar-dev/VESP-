# Contributing to VESP

Thank you for your interest in contributing to the Virtual Epileptic Patient pipeline! We welcome contributions from researchers, clinicians, and developers.

## Code of Conduct

Please treat everyone with respect and follow standard open-source etiquette.

## How to Contribute

1.  **Fork the Repository**: Create your own copy of the project.
2.  **Create a Branch**: `git checkout -b feature/AmazingFeature`
3.  **Commit Your Changes**: `git commit -m 'feat: add AmazingFeature'`
4.  **Push to the Branch**: `git push origin feature/AmazingFeature`
5.  **Open a Pull Request**: Submit your PR for review.

## Reporting Bugs

Please use the GitHub Issues tab to report bugs. Include:
- Description of the issue
- Steps to reproduce
- Expected behavior
- Screenshots (if applicable)

## Coding Standards

- **Python**: We follow PEP 8.
- **Type Hinting**: All public functions should have type hints.
- **Testing**: Ensure all new features are covered by tests (`pytest`).
